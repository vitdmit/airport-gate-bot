# Бот по вылетам и гейтам Москвы

Это бесплатный тестовый вариант для SVO, VKO и DME.

Идея простая: днем бот автоматически сохраняет live-гейты из табло, а в 09:00 по Москве собирает отчет за вчера. Это нужно потому, что часть аэропортов после вылета уже не показывает номер gate в публичной истории.

## Что будет в отчете

- аэропорт;
- ВВЛ / МВЛ;
- терминал;
- gate;
- количество рейсов с gate за сутки;
- расшифровка по каждому рейсу: время, авиакомпания, номер рейса, направление;
- кодшеринг считается как один операционный рейс, если совпали аэропорт, терминал, gate и время вылета.

Готовый файл появляется в папке `outputs`:

```text
outputs/gate_report_YYYY-MM-DD_daily.xlsx
```

## Как это работает

У системы два шага.

### 1. Сбор live-гейтов

Команда:

```powershell
python -m airport_gate_bot collect --airports SVO,VKO,DME
```

Она сохраняет маленькие JSON-снимки сюда:

```text
data/raw/YYYY-MM-DD/
```

Эти снимки нужны, чтобы не потерять gate, если аэропорт уберет его из истории после вылета.

Если в одном снимке gate был, а в более позднем снимке пропал, бот сохраняет найденный ранее gate и не затирает его пустым значением.

Для DME бот дополнительно пробует сверить ближайшие рейсы с официальным мобильным табло Домодедово. Это помогает поймать gate в короткое окно перед вылетом.

### 2. Отчет за вчера

Команда:

```powershell
python -m airport_gate_bot daily-report --date yesterday --airports SVO,VKO,DME
```

Для конкретной даты:

```powershell
python -m airport_gate_bot daily-report --date 2026-05-20 --airports SVO,VKO,DME
```

Отчет берет все не отмененные вылеты, которые бот видел в течение суток, и группирует их по gate. Для бесплатного режима это самый быстрый и устойчивый вариант.

Если нужна отдельная медленная сверка с внешним post-fact источником, есть команда:

```powershell
python -m airport_gate_bot verified-report --date yesterday --airports SVO,VKO,DME
```

Ее лучше использовать для контрольных проверок, а не как основной ежедневный бесплатный запуск.

## Первый важный момент

Полный отчет с gate получится только после того, как бот хотя бы сутки собирал live-снимки. Например:

- 20 мая бот весь день собирает данные;
- 21 мая в 09:00 он делает отчет за 20 мая.

Если запустить отчет за дату, когда бот еще не собирал live-снимки, часть gate будет пустой. Это нормально для первого теста.

Строго юридический "факт вылета" бесплатный live-режим не гарантирует на 100%. Поэтому первые несколько дней надо сверить отчет с ручными таблицами. Практически это хороший MVP: он ловит gate до того, как аэропорт убрал его из публичного табло.

## Бесплатный запуск без включенного компьютера

Самый простой вариант - GitHub Actions.

Что нужно сделать один раз:

1. Создать приватный репозиторий на GitHub.
2. Загрузить туда эту папку проекта.
3. Открыть вкладку `Actions` и разрешить запуск workflow, если GitHub спросит.

Дальше все работает само:

- каждый час GitHub запускает сбор live-гейтов;
- каждый день в 09:00 по Москве GitHub делает отчет за вчера;
- данные и отчеты сохраняются обратно в репозиторий;
- компьютер может быть выключен.

Расписание уже настроено в файле:

```text
.github/workflows/airport-gate-bot.yml
```

## Ручной тест на GitHub

В GitHub:

1. Открыть `Actions`.
2. Выбрать `airport-gate-bot`.
3. Нажать `Run workflow`.
4. Для `mode` выбрать:
   - `collect` - только собрать текущие live-гейты;
   - `report` - только собрать отчет за дату;
   - `all` - собрать live-гейты и сразу построить отчет.
5. В `report_date` оставить `yesterday` или указать дату, например `2026-05-20`.

## Локальный тест на компьютере

Если хочется проверить руками:

```powershell
python -m pip install -r requirements.txt
python -m airport_gate_bot collect --airports SVO,VKO,DME
python -m airport_gate_bot daily-report --date today --airports SVO,VKO,DME
```

Файл появится в `outputs`.

## Как читать качество отчета

В Excel есть лист `Качество`.

Там видно:

- сколько всего операционных вылетов найдено;
- сколько ВВЛ и МВЛ;
- сколько gate не найдено;
- сколько gate взято из live-снимков;
- сколько кодшерингов схлопнуто.

Лист `Рейсы` показывает строку каждого операционного рейса: точное время вылета, авиакомпания, номер рейса, направление, gate и источник gate.

Лист `Загрузка гейтов` - длинная таблица для графиков: один рейс = одна строка, есть gate, час, точное время, авиакомпания и направление.

Лист `Гейты x часы` - готовая почасовая сетка: строки это gate, столбцы это часы 00:00-23:00, значения это количество рейсов.

## Ограничение бесплатного варианта

Это MVP на публичных источниках. Он не является официальным договорным фидом аэропортов. Для боевого отчета после теста нужно сравнить несколько дней с ручными таблицами и понять процент покрытия gate по DME/VKO/SVO.

Если покрытие будет недостаточным, следующий шаг - либо добавить еще один live-источник, либо перейти на официальный/платный API.
